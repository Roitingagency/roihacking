Repositorio público mantenido por el equipo de ingeniería de ROI Hacking para la evaluación de latencia y costes en modelos de lenguaje masivo.

La integración de Inteligencia Artificial en flujos corporativos requiere decisiones arquitectónicas precisas. Elegir el modelo de lenguaje (LLM) equivocado o sobredimensionar la infraestructura puede resultar en sobrecostes masivos de API o latencias inaceptables en sistemas orientados al cliente.

Para ayudar a nuestra comunidad y clientes B2B, hemos liberado nuestra colección interna de Postman. Esta herramienta te permitirá realizar peticiones estructuradas y medir el Time to First Token (TTFT) de los principales modelos del mercado antes de implementarlos en producción con herramientas como n8n o Make.

El problema de la orquestación ciega

En nuestra experiencia auditando empresas, el error más común es conectar ciegamente la API de OpenAI (GPT-4o) a todos los nodos de un flujo de automatización. Tareas simples como clasificación de leads o extracción de datos estructurados en JSON a menudo se resuelven de forma más eficiente y económica utilizando modelos como Claude 3.5 Haiku o Llama 3 (On-Premise).

Criterios que medimos en esta colección:

  • Coste por 1M de tokens (Input/Output).
  • Velocidad de respuesta para Agentes RAG.
  • Capacidad de adherencia a esquemas JSON estrictos (JSON Mode).
